- **작성자**: Claude (재희님 요청)
- **버전**: v1.0

## 1. 개요

### 1-1. 목적

셀포스(Self-POS) 수동 테스트 시, 바코드 스캐너 장비 없이도 바코드를 빠르게 입력할 수 있는 크롬/엣지 확장 프로그램을 구현한다.

### 1-2. 문제 정의

| 현재 상황 | 목표 상황 |
|-----------|-----------|
| 바코드를 키보드로 직접 타이핑 | 바코드를 붙여넣기 → 버튼 클릭 한 번으로 완료 |
| 오타 발생 가능 | 정확한 바코드 자동 전송 |
| Playwright 자동화 코드에서만 사용 가능 | 수동 테스트 환경에서도 동일하게 사용 가능 |

### 1-3. 범위

- **In scope**: Chrome, Edge (Chromium 기반) 브라우저
- **Out of scope**: Firefox, Safari, 자동화 테스트 코드 수정

## 2. 기술 결정

### 2-1. Manifest V3 채택

Chrome과 Edge 모두 Manifest V3를 표준으로 채택함.  
Manifest V2는 2024년부터 Chrome에서 지원 종료 예정으로 V3 선택.

### 2-2. Content Script 방식

**대안 비교:**

| 방식 | 장점 | 단점 | 채택 여부 |
|------|------|------|-----------|
| Content Script | 페이지 DOM에 직접 접근, 키 이벤트 dispatch 가능 | — | ✅ **채택** |
| `chrome.debugger` API | CDP 직접 제어 (Playwright와 동일) | `debugger` 권한 필요, UX 방해 배너 표시 | ❌ |
| `execCommand('insertText')` | 입력창에 직접 텍스트 삽입 | 셀포스는 입력창 없이 body keydown 감지 | ❌ |

셀포스는 별도 input 요소 없이 **document body에서 keydown 이벤트를 감지**하는 방식이므로 Content Script에서 KeyboardEvent dispatch가 적합하다.

### 2-3. 이벤트 순서

Playwright `keyboard.type` 동작과 동일하게 맞춤:

```
각 문자:  keydown → keypress → keyup  (30ms 간격)
마지막:   keydown → keypress → keyup  (Enter)
```

## 3. 아키텍처

### 3-1. 컴포넌트 구성

```
┌─────────────────────────────────────────────┐
│               Chrome / Edge                 │
│                                             │
│  ┌─────────────────┐   ┌─────────────────┐  │
│  │   Popup UI      │   │  Content Script │  │
│  │  (popup.html)   │   │  (content.js)   │  │
│  │                 │   │                 │  │
│  │  [바코드 입력창]  │──▶│ KeyboardEvent   │  │
│  │  [스캔 버튼]     │   │ dispatch        │  │
│  │  [딜레이 슬라이더]│   │                 │  │
│  └─────────────────┘   └────────┬────────┘  │
│                                 │            │
│                    ┌────────────▼──────────┐ │
│                    │    셀포스 웹 페이지    │ │
│                    │  document.body         │ │
│                    │  keydown 이벤트 수신   │ │
│                    └───────────────────────┘ │
└─────────────────────────────────────────────┘
```

### 3-2. 메시지 흐름

```
popup.js
  → chrome.tabs.query({ active: true, currentWindow: true })
  → chrome.tabs.sendMessage(tabId, {
      type: 'SCAN_BARCODE',
      barcode: '8801234567890',
      delay: 30
    })
  ← { success: true } 또는 { success: false, error: '...' }

content.js
  → chrome.runtime.onMessage.addListener()
  → typeBarcode(barcode, delay)
    → for each char: dispatchKeyEvents(target, char)
    → sleep(delay)
    → dispatchKeyEvents(target, 'Enter')
  → sendResponse({ success: true })
```

## 4. UI 설계

### 4-1. 팝업 레이아웃

```
┌──────────────────────────────┐
│  🔲  Barcode Scanner         │  ← 헤더
├──────────────────────────────┤
│                              │
│  ┌──────────────────────┐    │
│  │ 바코드를 붙여넣으세요  │    │  ← textarea (붙여넣기 자동 포커스)
│  └──────────────────────┘    │
│                              │
│  딜레이: [──●──────] 30ms    │  ← 딜레이 슬라이더 (10~200ms)
│                              │
│  [      스캔 (Enter)     ]   │  ← 스캔 버튼
│                              │
│  ✅ 스캔 완료!               │  ← 상태 메시지 (성공/실패)
└──────────────────────────────┘
   너비: 320px
```

### 4-2. UX 정의

| 동작 | 반응 |
|------|------|
| 팝업 열기 | textarea 자동 포커스 |
| textarea에서 Enter 키 | 스캔 실행 |
| 스캔 버튼 클릭 | 스캔 실행 |
| 빈 값 제출 | "바코드를 입력해주세요" 경고 |
| 스캔 성공 | "✅ 스캔 완료!" 1.5초 표시 후 초기화 |
| 스캔 실패 | "❌ 실패: {에러 메시지}" 표시 |
| 다줄 입력 (여러 바코드) | 첫 번째 줄만 사용 (trim 처리) |

## 6. 보안 및 권한 최소화

| 권한 | 이유 |
|------|------|
| `activeTab` | 현재 탭에만 content script 메시지 전송 |
| `scripting` | content script 동적 주입 (필요 시) |
| `<all_urls>` (content script) | 셀포스 URL이 환경마다 다르므로 제한 없이 적용 |

불필요한 권한 없음. `storage`, `tabs`, `cookies` 등 사용 안 함.

## 7. 제약 사항 및 알려진 한계

| 항목 | 내용 |
|------|------|
| React 합성 이벤트 | DOM KeyboardEvent로 동작. onChange 연결 input이 있으면 미동작 가능 |
| CSP(Content Security Policy) | 일부 사이트에서 content script 실행 차단 가능 (셀포스는 해당 없음) |
| 팝업 포커스 이탈 | 팝업을 닫아야 셀포스 페이지가 포커스됨 — 자동 닫기 처리 |
| 다중 탭 | 현재 활성 탭에만 전송 |

## 8. 테스트 계획

1. 확장 프로그램 로드 (개발자 모드)
2. 셀포스 페이지 접속
3. 팝업 열기 → 바코드 붙여넣기 → 스캔 버튼 클릭
4. 상품이 정상 스캔되는지 확인
5. 다양한 바코드(8자리, 13자리) 테스트
6. Edge에서도 동일하게 테스트
